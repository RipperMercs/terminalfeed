What Is URL Encoding?

URL encoding (also called percent encoding) replaces unsafe characters in a URL with a % followed by two hexadecimal digits. For example, a space becomes %20, and @ becomes %40. This ensures URLs are valid and can be safely transmitted over the internet, since URLs can only contain a limited set of ASCII characters.

When Do You Need URL Encoding?

You need URL encoding when building query strings with user input, constructing API URLs with special characters, encoding redirect URLs, building OAuth callback URLs, or any time you need to include characters like spaces, ampersands, or hash symbols in a URL parameter.

encodeURIComponent vs encodeURI

encodeURIComponent() encodes everything except A-Z a-z 0-9 - _ . ! ~ * ' ( ). It's used for encoding individual URL parameters. encodeURI() preserves characters that have special meaning in URLs (like :, /, ?, #). This tool uses encodeURIComponent for maximum safety.
